# Public-Safe Contribution Rules

This repo is for practical automation guidance, not private implementation dumps.

## Accepted

- Public product docs, open-source repos, pricing pages, and broadly available templates.
- Workflow recipes written with fictional businesses, fictional contacts, and generic sample data.
- Clear notes about when a tool needs paid setup, API keys, or account verification.

## Not Accepted

- Client exports, private screenshots, private CRM data, real customer names, API keys, passwords, tokens, or internal URLs.
- Affiliate-only link farms or generic “AI tool” lists without a small-business workflow use case.
- Instructions that require unsafe scraping, spam, platform abuse, or undisclosed outbound automation.

## Review Checklist

- The item solves a concrete workflow problem.
- The item is understandable to a small-business operator or freelancer.
- The item has a public URL and a maintenance signal.
- The notes explain the cost/friction level honestly.
